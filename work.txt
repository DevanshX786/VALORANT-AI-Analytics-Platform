[5] STEP 3 SCORING LOGIC DEFINITIONS (Validated in notebook 02_scoring_logic.py)
These are the mathematical formulas we will test in `notebooks/02_scoring_logic.py`:

1. Mechanical Score (Raw Firepower)
   - logic: (Average Combat Score * 0.45) + ((Kills / max(Deaths, 1)) * 30) + ((2k*2 + 3k*3 + 4k*4 + 5k*5) * 2)
   - goal: Reward high-impact fragging and multi-kills.

2. Clutch Score (Under Pressure)
   - logic: (1v1 * 1) + (1v2 * 2) + (1v3 * 4) + (1v4 * 8) + (1v5 * 15)
   - goal: Exponential reward for higher-difficulty situational clutches.

3. Entry Success (Opening Duels)
   - logic: First Kills / max((First Kills + First Deaths), 1) + (First Kills - Deaths)
   - goal: Identify duelists/initiators who find the opening breakthrough without dying first.

4. Utility Usage Score (Support Impact)
   - logic: Assists per round + (Spike Plants * 0.5) + (Spike Defuses * 0.5)
   - goal: Measure supportive contribution beyond pure aim.

5. Economic Efficiency (Value generation)
   - logic: (We will check if Econ Rating exists; otherwise, default to standard KAST/Rating heuristics).
   - goal: Ability to thrive on low-buy/eco rounds.

6. Consistency Score (Variance)
   - logic: 1 / (Standard Deviation of `Rating` over last N matches + 0.1)
   - goal: Punish coin-flip players; reward stable baselines across tournaments.

7. Map/Agent Versatility
   - logic: Distinct Count of Agents played + Win Rate spread across Maps.
   - goal: Reward flex players over one-tricks.

[6] STEP 4 - BASELINE MODEL TRAINING (src/models/baseline_model.py)
Status: CURRENT

GOAL: Train an untuned XGBoost classifier that predicts Match Win/Loss
using aggregated player scores as team-level features.

--- HOW IT WORKS ---

Step A: Aggregate Player Scores into Team Strength
  - For each match + team, pick the 5 players that played (from overview).
  - For each player, compute their per-match scores using PlayerScorer.
  - Aggregate the 5 players into team-level stats:
      * mean(Mechanical_Score)  -> avg raw firepower of the team
      * mean(Clutch_Score)      -> avg clutch capability
      * mean(Entry_Success)     -> avg opening duel performance
      * mean(Utility_Score)     -> avg support/utility contribution
      * max(Mechanical_Score)   -> star player headroom (ceiling fragging)
      * std(Mechanical_Score)   -> team balance (low std = even carry load)

Step B: Build a Head-to-Head Feature Row
  - Per match row: [TeamA_features] - [TeamB_features] = DIFF features
  - This creates a single row per match: how much stronger is Team A vs B?
  - Target (label): 1 if Team A won, 0 if Team A lost.
  - Source for labels: matches/scores.csv (compare Team A Score vs Team B Score).

Step C: Train XGBoost Baseline
  - Split: 80% train, 20% test (be careful NOT to leak future matches into train).
  - Model: XGBoostClassifier with default hyperparameters (untuned baseline).
  - Metrics: Accuracy, AUC-ROC, Log Loss.
  - Save the trained model to: models/baseline_xgb.json

Step D: Output
  - Print evaluation metrics (Accuracy, AUC).
  - Plot feature importance to show which pillar drives win prediction most.

--- KEY FILES ---
  - src/models/baseline_model.py  -> Main training pipeline
  - notebooks/03_baseline_model.py -> Prototyping sandbox
  - models/baseline_xgb.json      -> Saved model artifact
